## 1. C ABI сигнатуры пяти entry points

### Forward
Файл: `libs/flash_attention_v121r_diet_hd128_fp8_forward.cu`  
Kernel: `fa96b_kernel` (namespace-less; namespace-less вызов через `launch_fa` helpers).

```c
__global__ void __launch_bounds__(FA_THREADS, 2)
fa96b_kernel(
    const uint8_t *__restrict__ Q,      // FP8 e4m3 [bh, sl, hd]
    const uint8_t *__restrict__ K,      // FP8 e4m3 [bh, sl, hd]
    const uint8_t *__restrict__ V,      // FP8 e4m3 [bh, sl, hd]
    __half        *__restrict__ O,      // FP16 [bh, sl, hd]
    int seq_len, int head_dim, int causal,
    float scale,                        // 1/sqrt(hd), passed pre-mul
    float qk_descale, float v_descale,  // FP8→FP16 rescale (обычно 1.0)
    int window);                        // 0 = full/causal, >0 = sliding
```
Grid: `bh * ceil(sl/FA_BR)` (FA_BR = 128 в v121r). Block: 128 threads (4 warps).

### D-precompute (dK/dV pre-step)
Файл: `libs/fa_bwd_dk.cu`  
Namespace: `fa_bwd_dk`

```c
void launch_d_precompute(
    const __half *O,   // fwd O output
    const __half *dO,  // upstream gradient
    float        *D,   // output D[bh, sl] = row-sum(O · dO)
    int bh, int sl, int hd, cudaStream_t stream);
```
D формат: `float [bh, sl]`, row-major (dim0=batch-head, dim1=seq).

### Что fwd отдаёт для bwd
- **O**: FP16 attention output (для D-precompute — суммируем строчку `O·dO`).
- **L**: **НЕ виден в текущей fwd ABI** (`fa96b_kernel` возвращает только `O`). Требуется расширение fwd:
  - вариант A: добавить `float *L` output в kernel signature
  - вариант B: recompute L из первого pass bwd (дороже)
  - **гейт W1**: доработать fwd отдать L. Не готово в текущем состоянии.
- Формат L: ожидается `float [bh, sl]`, row-major.

## 2. Скейл-схема FP8

### Квантизация Q/K/V (вход fwd)
- Тип: **FP8 e4m3** (7-битная мантисса + знак; range ≈ ±448)
- Схема: **per-tensor scalar** (fwd подписывает `qk_descale`, `v_descale` — оба обычно 1.0)
- Квантизация делается **вне ядра** (fwd/bwd видят уже-FP8 тензоры)
- Функция host-quant: `float_to_e4m3_host()` в `fa_bwd_common.cuh`

### Деквантизация в bwd
- **dQ**: MMA-C `dQ_acc += dS · K` в FP32 acc (K FP8 → mma умножитель принимает FP8 native)
- **dK**: MMA `dK_acc += dP^T · Q` в FP32 (Q FP8 → native)
- **dV**: MMA `dV_acc += P^T · dO` — P FP8, dO FP16 (V-side dequant в scale computation)
- **Все bwd выходы**: FP32 (`float *dQ/dK/dV`)
- **v_descale/qk_descale**: не используются в bwd (dO FP16 масштабируется через L и scale)

### Наблюдения
- Текущая схема — **static per-tensor descale=1.0**. Это простой FP8 без scale-обучения.
- Для полноценного FP8-train потребуется **per-block или per-channel scale**: расширить fwd/bwd API `qk_scale[bh]`, `v_scale[bh]` (или tile-level).
- **Стохастическая квантизация** и **saturating cast** не установлены — гейт W2.

## 3. Произвольный sl + causal-флаг из ABI подтверждён?

- `int seq_len` в fwd + `int sl` в bwd — **произвольный** (не константа времени компиляции).
- `int causal` (0/1) + `int window` (0 = full/causal, >0 = sliding) — оба **runtime**.
- Кернели корректно обрабатывают:
  - `sl` не кратный `Br=64` (last-tile OOB маскировка через `i_g >= sl`)
  - `causal=1` — kt_end = qt+1
  - `window > 0` — маска `j_g < i_g + 1 - window`
- **Форма верифицирована** в тестах через 11-form harness + F9/F10 (sl=2048).

## 4. PyTorch в контейнере

- **Python**: `/usr/bin/python3` = **Python 3.10.12**
- **PyTorch**: `2.8.0+cu128` (cu128 wheel = CUDA 12.8 runtime)
- **CUDA visible**: `torch.cuda.is_available() = True`, `device_count = 1`
- **Device**: NVIDIA RTX PRO 6000 Blackwell Workstation Edition (**CC 12.0 = sm_120**)
- **VRAM**: 102.0 GB (**HBM3** predsumably)
- **Компилятор**: `/usr/local/cuda-13.1/bin/nvcc` = **CUDA Toolkit 13.1**
  - **Расхождение**: nvcc 13.1 vs torch cu128 (12.8). PyTorch bindings/extensions скомпилированные под nvcc 13.1 могут потребовать cu13-wheel torch. Проверка при W1 сборке.
