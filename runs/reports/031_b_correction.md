## 2. Изъятие E2E-прогноза (по Vugar-приёмке)

### 2.1 Строка изымается из 031

> ~~"(a) → −35-45% E2E"~~

**Не обосновано bandwidth-моделью**:
- merged wall НЕ bandwidth-bound (BW-util 33.6%, ratio 2.97× floor)
- Wall определяется **внутренностями** (mio, short_sb, барьеры) — из прежних отчётов merged mio ~24-30%, short_sb ~10-15%, барьеры несколько
- **Bandwidth-модель для выигрыша merged неприменима** — уменьшение DRAM не переводится в wall в пропорции

### 2.2 Реальная база для прогноза (для фазы 032 п.5)

E2E-прогноз может появиться **только** из суммы:
1. **Счёт вырезаемых MIO-ops merged**: сколько STS.U8/STS.32 exit ops убирается вместе с dS_T scatter (по SASS-счёту)
2. **Счёт вырезаемых барьеров merged**: смещение sync-точек после устранения T-staging
3. **SMEM-дельта merged**: -5120 B/block от снятия smdS_T буфера — может добавить occupancy, если +1 block/SM возможен
4. **DRAM-дельта цепи**: -8 GB dS_T write в merged, минус возможные L2 hits на dS_nat второго читателя
5. **Цена транспонирования в dk**: SMEM или регистровое, + MIO-ops load pattern

**Прогноз конкретных цифр** — работа фазы 032 п.5.
