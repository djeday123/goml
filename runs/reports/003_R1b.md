### Root cause identified — Q layout несогласован с MMA

MMA `mma.sync.aligned.m16n8k32.row.col.f32.e4m3.e4m3.f32`:
- A row-major `[M=j][K=i]` (dS^T читается корректно из smdS_T stride Br)
- **B col-major `[K=i][N=d]` — тут ошибка**

В моём коде: `uint32_t B0 = *smQ[k_i_lo * Hd + n_d * 4]` — это чтение Q **row-major [i][d]** напрямую. Но MMA требует B col-major [i][d] — накладываемый в память как `Q_T[d][i]`.

**Sealed dK делает phase транспонирования** smQ → smQ_T через explicit transpose с stride QT_STRIDE=68 (все warps LDS + BARRIER + STS). Моё kernel_dk_new пропустило эту фазу — Q читается как есть, layout mismatched → MMA генерирует mangled bytes → dK ≠ sealed.

Также misaligned address появляется из-за read pattern, где `k_i_lo*Hd + n_d*4` при `k_i_lo=48` даёт offset 6144+n_d×4, but reading uint32_t at even byte offset that isn't multiple of 4 → misaligned. Actually 48*128 = 6144 (mult of 4), + n_d*4 (mult of 4) → aligned. Should be OK. Misaligned может быть из другого места — например epilogue write pattern.

### Fix план

1. Реализовать phase транспонирования smQ → smQ_T (mirror sealed dK).
   - Читать Q rows в smQ (как сейчас).
   - BARRIER.
   - All-warps transpose smQ→smQ_T с stride QT_STRIDE=68 (bank-safe).
   - BARRIER.
   - MMA B читает smQ_T как col-major [i][d].
2. Alternative: LDG Q напрямую в smQ_T layout (транспонированный write from cp.async source).
3. Дополнительно: проверить misaligned address в epilogue write pattern.

Fix — задача R1b-fix, вне текущего цикла. R1c-ТЗ НЕ начинать до 004_R1b_fix.md с BIT-EXACT PASS.
