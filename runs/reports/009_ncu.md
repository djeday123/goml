## 009-0.3 — Разбор +61r сверх сметы (192 → 253)

### Проекция vs факт по стадиям
Проекция была на «пиковую» live-set. Факт — pt xas выделяет под max **live-range union** через весь qt-body:

| Фаза qt-цикла | Live regs (проекция) | Live regs (SASS-факт по nvcc) |
|--------------|:--------------------:|:-----------------------------:|
| MMA-A end | Qr(16) + Sr(16) + dV_acc(64) + const(15) | ~111 |
| Softmax done | + Pr(16); Sr released; halves scratch | ~127 |
| MMA-B done | + dPr(32); Qr released | ~143 |
| **dS jam-x2 peak** | Pr(16)+dPr(32)+dV_acc(64)+halves+intermediates | **~180-200** |
| Drain phase | Pr(16)+dV_acc(64)+chunk regs | ~90 |
| STS smP_T | Pr(16)+dV_acc(64)+halves | ~90 |
| MMA_dV peak | Ar(4)+Br0/Br1+dV_acc(64) | ~72 |

Ptxas allocates for the MAX peak in ANY SASS block. Плюс:
- **Persistent address bases** (Qb, Kb, Vb, dOb, dS_nat_out, dS_T_out, smK, smV, smQ_region, smdO base, smL/smD, smdS_T_stage) — 10+ pointers × 2 regs (64-bit) = ~20 regs unused between phases
- **Swizzle constants** (k_xor, dO_xor_el, PT_xor_*) — few regs
- **Compiler ILP unroll** (fully-unrolled `#pragma unroll` on jam-x2 + Q·K^T ks-loop + MMA_dV kb×ni) увеличивает intermediate live-set

Merge-налог на live-ranges: **фазы t6 (MMA-B), t7 (dS compute), t8-9 (scatter+drain) все одновременно живы для Pr + dPr + dV_acc = 112 регов accumulators.** Плюс intermediate halves + fp8x2 packing. Плюс addressbases.

**+61 регистр = стоимость соединения жизненных диапазонов, не bug**. spill=0, 2 blocks/SM держатся.
