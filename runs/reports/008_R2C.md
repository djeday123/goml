## Стройка (008-1)

**KV-owned merged**: `grid = bh × n_kt`, K/V резидентны, `for qt` inner-loop.

### Барьеры per qt (6, как Vugar-правка)
- **t3** post-loads (после cp.async Q/dO/L/D wait)
- **t_new1** post-smQ-reads pre-scatter (гонка warp-скью через байтовое перекрытие невыровненных оверлеев smdS_stage поверх smQ)
- **t9** pre-drain
- **t_new2** post-drain pre-P_T STS
- **t11** pre-MMA_dV
- **t13** end qt (перед next cp.async Q overwrite of smQ_region)

### Bit-exact критические инварианты (заложены в код)
1. **Порядок qt-цикла** == sealed dV_p1 (causal: `qt_start = causal ? kt : 0`; loop `for qt = qt_start; qt < n_qt`).
2. **MMA_dV**: `for kb=0..KB_DV=4; for ni=0..NI_DV=16` accumulation order == sealed lines 288-327. fp32 dV_acc non-assoc → order == bytes.
3. **smP_T layout**: fp16 halves h_p00/01/10/11 STS'd со SEALED XORs (`PT_xor_even_wr = l_mod4<<4`, `PT_xor_odd = +8`) lines 279-282.
4. **dS quant path** == R1a ds_gen: `pa_mlo_nlo * (dPr[ni_a][0] - D_lo)` → halves2half2 → fp16x2_to_e4m3x2 (unchanged jam-x2).
5. **dS_nat / dS_T тайл-локальны** (правка Vugar-3): байты at (i, j) зависят только от Q[i], K[j], V[j], dO[i], L[i], D[i] — kросс-тайлового состояния нет → KV-ownership не тронул байты.
