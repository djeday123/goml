### §5.d Техлог-сага кампании 040-061 (LDSM-эпоха)

**Строгий поток**:

1. **040 (LDSM class #7)**: перевод dV MMA-B smdO reader (256 LDS.U16 → 32 LDSM.x4.trans.b16). ABBA median **−12.28% wall**. Единогласный 8/8. Первый успешный reader-only LDSM в кампании.

2. **041 (dq_new разморозка)**: разморозка `d5lite_pack_pi` (69r vs 56r prod). ABBA **−3.47%** wall единогласно. Неожиданный KEEP (ожидание было ~−1.5%).

3. **War за wait (052/053/054/055/056)**: 5 механизмов, ВСЕ красные — техпаспорт равновесного ядра. Ярлык: "merged v40 равновесное ядро; wait 33% не сдался ни prefetch (Q/dO), ни LDS-ремеслу, ни L2-hint, ни LDG-регистру. Двигается только форматом dO (FP4) или persist."

4. **057 стенд-реатрибуция**: Zombie процесс r2c_merged_wall PID 840120 (6h49m Rl, 1434% CPU) от background запуска в окне 050-055 объяснил x2-сдвиг baseline (~44 → ~86 ms). Kill → возврат к ledger. Замки стенда наследие (a/b/c/d): расширенный gate с foreign-check, quiet_gpu.sh, autostart инвентарь, zombie source атрибуция.

5. **S2v4-сага**:
   - **051 автопсия S2v3**: bank storm ×5.07 LD conflicts (32-way, row-stride 128B без свизла); судья 044 §5.f опровергнут.
   - **058 бумага**: Кандидат B (swz_byte) выбран; судья ДВУХ фаз ЗЕЛЁНЫЙ (writer 8-way, reader 4-way structural).
   - **060 мост**: маркер агента 058 отвергнут перебором (8192 collisions); row-only marker + LDSM.x2.trans.b8 на свизлованном mock **100% (16384/16384 + 64/64 rows)**.
   - **061 col-проход + production**: byte ORDER = MMA-B fragment (R0 uniform + R2=R0 dup ISA-045); правка dk_new с смертью Q_T pack (feeder + 12 SHFL + 16 STS + π_V + smQ_T 8704B + барьер line 310); ABBA **dk −19.24% / E2E −4.48% KEEP** единогласно; RACECHECK 0 ✓; шторм 051 НЕ вернулся (**events −57.8%, waves −24.2%, mio −15.5pp**).

**Захоронения кампании с причинами**:
- L2 persist window (044-047): 4 режима dead, dk DRAM 68 MiB ≫ 20 MiB threshold
- S2v2 (048): регистровый wall 161r/3blk (R2/R3 duplicates ISA quirk)
- Marker aliasing (049): `(row<<4)|(col&0xF)` for row≥16
- S2v3 (050-051): bank storm 32-way (row-stride 128B без свизла)
- smQ-prefetch (052): barrier +2.29pp + ST +123% съедают
- dO half-prefetch (053): split-buffer сам новая яма +1.895%
- M5+A' LDS (054): sub-2%, мосты #5/#6 deferred
- L2-hint (055/056 A): +1.99% per-op patologiчна на sm_120a
- LDG-регистр (055/056 B): +1.53% STS bank retry
- Маркер 058 (агентская формула): 8192 collisions

**Уроки свода** (для будущих кампаний):
1. **Судья ДВУХ фаз** для кооперативных LDSM (address + data phase раздельно, свод 052)
2. **Инъективность маркеров обязательна ДО использования** (049/058/060, перебором)
3. **Гейт-тишина**: foreign compute-apps abort до замеров (057 замки)
4. **Курс недооценивает сериализованные сносы**: линейный 0.0479%/op занижает fp8 LDSM sequence (0.4-1% ошибка per structural cost)
5. **Racecheck обязателен когда барьер умирает** (правило 13, применено 052/056/061)
6. **Свизл писателя лечит адресную фазу**: 32-way storm (row-stride 128B без свизла) → 4-way structural пол при row-based XOR (051 → 061)
