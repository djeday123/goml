## Дизайн

kernel_dq_new: **потребитель dS_nat** (материализованный ds_gen'ом), выдающий gradient dQ.

- **Grid**: `bh × n_qt` (block owns Q-tile), симметрично dk_new.
- **MMA-C**: `dQ_acc += dS · K_T` (m16n8k32 row.col, FP8×FP8→**FP16 acc**, packed uint32[16][2]).
- **K_T-фаза**: зеркалом sealed AA1 phase 1.5 — cp.async K swizzled → LDS read swizzled → STS natural к_T (aliased overwrite) со STRIDE=68.
- **dS-load**: cp.async 16-байт из global dS_nat со **stride_ds = (sl+15)&~15** (ABI от 005a), в smdS со **SMDS_STRIDE=80** (16-aligned для cp.async dest + bank-conflict-free MMA read).

### BIT-EXACT инварианты vs sealed AA1
1. dQ_acc = FP16x2 packed [16][2] (тот же уникод, тот же descale-эпилог)
2. **MMA-порядок: `kt` outer → `kb` inner → `ni` innermost** — идентично sealed AA1 линии 505-529. fp16-acc **не ассоциативно** → тот же порядок = те же биты
3. K load swizzle XOR `(j_local & 7) << 4` — mirror sealed line 256
4. K_T read XOR `l_div4 << 4` — mirror sealed line 485
5. K_T write natural stride 68 — mirror sealed
6. Epilogue unpack packed fp16 → fp32 → *scale → store — mirror sealed lines 548-565

SMDS_STRIDE=80 vs sealed 68: значение fp8 на позиции (i,j) одно и то же; MMA читает те же 4 байта → бит-в-бит совпадает арифметика. Stride влияет только на SMEM layout.
