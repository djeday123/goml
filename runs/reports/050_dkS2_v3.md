## §0. Леджер-правки (бумага)

### 0.a Аннулирование вердикта 048 §3

**Прежний вердикт (048 §3)**: "R0hi==R0lo — LDSM cooperative layout не даёт b1-pair через row+16 offset. Pack Q_T неизбежен для sm_120a m16n8k32 fp8."

**АННУЛИРУЕТСЯ**: наблюдение "R0hi==R0lo" под алиасирующим маркером `(row<<4)|(col&0xF)` **неразличимо от успешной доставки b1**. Для row=16: `(16<<4)|c = 0x10c → uint8_t = 0x0c` — совпадает с row=0 марком → aliasing.

**Морозильная бирка "Pack Q_T неизбежен" СНЯТА**.

### 0.b Утверждение "формулы 048 и 049 эквивалентны" — paper-unverified

Помечено paper-unverified до §1 (мост v2).

### 0.c Новое правило свода

**Инъективность маркера**: домен координат и разрядность маркера доказываются ПЕРВОЙ СТРОКОЙ любой маркер-пробы. Пробы с алиасирующим маркером **ничтожны**.

## §1. Мост v2 (до правки, обязательный)

Файл: `libs/probe_bridge_v2_050.cu`. Инъективный маркер `byte = row & 0x3F` (6 бит на 64 rows — совпадает с доменом). Полное покрытие: **kb ∈ [0..1] × np ∈ [0..7] × lane ∈ [0..31] × 4 регистра = 4096 значений (× 4 байта per uint32 доступны для проверки, тестировано 4096 byte-positions per формула)**.

### Результаты

```
=== Формула A (base-offset 16+lane) ===
Match: 4096 / 4096 (100.00%)
Sample kb=0 np=0 lane 0: R0lo=03020100 R1lo=03020100 R0hi=13121110 R1hi=13121110

=== Формула B (lane-shift (lane&15)+16) ===
Match: 4096 / 4096 (100.00%)
Sample kb=0 np=0 lane 0: R0lo=03020100 R1lo=03020100 R0hi=13121110 R1hi=13121110

=== OOB-дискриминатор ===
Lanes 0..15 R0/R1 diff (normal vs OOB): 0 / 32 slots
Вердикт OOB: HW ИГНОРИРУЕТ лейны 16..31 — production-формула = (A) или (B) эквивалентны
```

**Обе формулы A и B доставляют b1 корректно** ✓ (rows 16..19 в R0hi, rows 24..27 в R1hi для lane 0). Sample R0hi = 0x13121110 = rows {16,17,18,19} — b1 ✓.

### OOB-дискриминатор — вердикт

**HW ИГНОРИРУЕТ лейны 16..31** для output R0/R1: подача чужих строк (row=0) лейнам 16..31 при формуле A **не изменила R0/R1 лейнов 0..15** (diff=0).

**Формулы A и B эквивалентны** для tile 0 output. **Выбор production**: **Формула B** (`(lane&15)+16`) — in-bounds по построению на всех kb (для kb=0: rows 16..31; для kb=1: rows 48..63; никогда OOB). Формула A требовала бы clamp для kb=1 (rows 64..79 = OOB).

### Ирония 048 — задокументирована

**Формула 048 `(lane&15)+16` была КОРРЕКТНОЙ**. 048 FAIL был из-за marker aliasing в микропробе, не layout. Sealed 048 морозильная бирка снята (§0.a).

## §4. Морозилка dk_new S2v3 с честной биркой

По TZ §7 "При красном после переработки: морозилка с liveness-таблицей и ЧЕСТНОЙ биркой".

### Честная бирка

> **«ISA-доставка b1 доказана (мост v2 §1: обе формулы 100% match, OOB HW игнорирует lanes 16..31). Регистровая стена решена (101r vs 128r baseline, 5 blk bonus). ОДНАКО wall +118% регрессия — LDSM.x2.trans.b8 pathway фундаментально МЕДЛЕННЕЕ чем sealed pack Q_T pathway. Причина: LDSM per-instruction latency + 2 LDSM per (kb, np) в hot loop stall'ят больше чем 64 LDS.U32 + предвычисленный pack.»**

### Liveness-таблица факта

| Компонент | Baseline (033 sealed) | S2v3 attempt | Delta |
|:--|:-:|:-:|:-:|
| **regs** | 128 | **101** | **−27** (5 blk bonus) |
| SMEM | 20992 B | **12288 B** | −8704 |
| Blocks/SM | 4 | **5** | +1 (бонус) |
| B-op path | 64 LDS.U32 (packed Q_T) | 32 LDSM.x2 (natural Q) | net −ops |
| **Wall dk isolated** | ~18.9 ms | **~41.2 ms** | **+22.3 ms (+118%)** |
| Ops (feeder + pack) | 16 LDS + 12 SHFL + 16 STS + 64 PRMT | 0 | −108 |
| Ops (B-op MMA read) | 64 LDS.U32 | 32 LDSM.x2 | −32 |

**Ключевой блокер (единственный реальный)**:
LDSM.x2.trans.b8 **per-instruction cost** (включая cooperative fetch overhead) существенно выше чем 2 LDS.U32. Даже с net −ops (16+12+16 = 44 общих ops сэкономлены), 32 LDSM в hot loop создают dep-chain, который MMA не может скрыть.
