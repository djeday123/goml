**Δ streams vs sequential = -0.085 ms (-0.14%) — NOISE. Реального overlap НЕТ.**

### Атрибуция contention (Vugar-предсказан)
- ds_gen grid = bh × n_qt = 128 × 128 = 16384 blocks (2 blocks/SM = 8-wave residency)
- dV grid  = 16384 blocks (аналогично)
- dk_new, dq_new grids = 128 × 128 = 16384 blocks каждый
- 188 SMs × 2-3 blocks/SM = 376-565 concurrent slots
- Все ядра заполняют GPU до макса — streams только очередят волны, не параллелят их
- Проекция 30.35 ms = **max()-оптимизм** (жил в мире свободного GPU); реально contention на тензор-пайпе + L2 съедают весь ожидаемый overlap

Streams-win может быть реален только для не-canonical форм (маленькие bh/sl, где grid < SM_count × block_limit).

## FLOPS conventions — ОБЕ

Base (fma-count normalized): `base = 2 · bh · sl² · hd · cf`
Каноник bh=128, sl=8192, hd=128, cf=1.0 → **base = 1.0995e12 FMA**.

| Convention | FLOPS | Sequential (59.31 ms) | Streams (59.23 ms) | Sealed ref (61.6 ms) |
|-----------|:-----:|:---------------------:|:-------------------:|:--------------------:|
| **16N²d** (Tri Dao Variant 3, executed sealed-ref для прогрессии) | 16 · base = 17.59e12 | **297.83 T** | 298.22 T | 285.44 T |
| **12N²d** (R1 actual executed: 6 MMA = 12N²d) | 12 · base = 13.19e12 | **222.62 T** | 222.92 T | (n/a — sealed делает 8 MMA=16N²d) |
| **10N²d** (fused-ideal лимит, "если бы все MMA слились") | 10 · base = 11.00e12 | 185.51 T | 185.76 T | (n/a) |

### Что R1 действительно исполняет — 6 матмулов:
1. ds_gen: Q·K^T (recompute S)
2. ds_gen: dO·V^T (compute dP)
3. dV_p1: Q·K^T (второй recompute S — независимая cudaMalloc-путь, sealed unchanged)
4. dV_p1: P^T·dO
5. dk_new: dS^T·Q
6. dq_new: dS·K
= **6 MMA total (12N²d executed FLOPS)**

Sealed делает 8 MMA (16N²d): 3× Q·K^T (dQ/dK/dV) + 2× dO·V^T (dQ/dK) + P^T·dO + dS^T·Q + dS·K.

**R1 сократил на 2 MMA (25% меньше compute), но wall упал только на 3.72% — memory-bound регион, MMA-flop-savings мало конвертируются.**

## Вилка дальше (R2C = слом 6 → 5 MMA)

### Наблюдения на числах:
- ds_gen 20.77 ms и dV 19.77 ms **обе делают Q·K^T + рабочий MMA** — суммарно 40.5 ms, доминантные затраты.
- Если слить [ds_gen + dV_p1] → merged kernel:
  - Общий S = Q·K^T recompute **экономится один раз** (2 MMA → 1 MMA)
  - Slot один kernel: регистры ~130 (ds_gen) + dV_acc 64 → ~200r/0s ожидаемо
  - SMEM смета: ds_gen 38.9KB + dV_specific (V + dV_acc); нужен профиль
- R2C ожидание: `merged(ds_gen + dV) → 25-30 ms` + dk_new 9.42 + dq_new 8.58 → E2E ~44-48 ms, streams-projection ниже
- Резерв 3-block occupancy умирает при 200r → 2 blocks/SM (унаследовано от sealed dV)

### Non-canonical (streams-win-territory):
- Малые формы (sl ≤ 1024) — grid не заполняет GPU → streams должны дать real overlap. Не измерено в этом отчёте (канон-only). Отдельная задача.
